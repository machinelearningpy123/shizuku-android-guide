Shizuku FAQ

These are the short answers that matter most before readers move from installation into actual use.

What does Shizuku actually do? +

Shizuku is a bridge that lets compatible Android apps access certain system APIs through an adb or root-backed service. It is not a one-tap optimizer by itself. Its value comes from the apps that integrate with it.

Is Shizuku only for rooted phones? +

No. Shizuku can work through wireless debugging or adb setup without root, and the official release notes for version 13.6.0 also mention improved auto-start behavior without root on Android 13+ when the device is connected to a trusted WLAN.

What is new in Shizuku 13.6.0? +

The official GitHub release notes say version 13.6.0 adds support for Android 16 QPR1, updates the start command, and supports auto start without root on Android 13+ when connected to a trusted WLAN.

What should I check before using Shizuku with another app? +

First make sure the package is really moe.shizuku.privileged.api, then use the official setup guide, and only grant Shizuku-backed access to apps you actually trust because those apps can request elevated system capabilities.
